✅ Java Interface - Interview Answer:

In Java, an interface is a reference type that acts as a contract for classes. It contains abstract methods (by default, public and abstract) and static/final variables. From Java 8, interfaces can also have default and static methods, and from Java 9, private methods as well.

A class implements an interface to provide the implementation of its declared methods.

🔹 Key Points:

  • Used for abstraction – defines what a class must do, but not how.

  • Supports multiple inheritance (Java doesn’t allow multiple class inheritance but allows a class to implement multiple interfaces).

  • Interface methods are implicitly public and abstract (except default, static, and private).

  • Variables are public, static, and final by default.

🔹 Syntax Example:

java
interface Vehicle {
    void start(); // abstract method
}

class Car implements Vehicle {
    public void start() {
        System.out.println("Car is starting");
    }
}

🔹 Advanced Features:

  • Default methods (Java 8): Provide method body in interface.

  • Static methods: Belong to interface, not the instance.

  • Private methods (Java 9): Used internally within default or static methods.

🔹 Use Cases:

  • Achieving loose coupling.

  • Supporting polymorphism.

  • Defining common behavior across unrelated classes.

🔚 Summary:

An interface in Java defines a blueprint of behavior without implementation. It supports abstraction, polymorphism, and multiple inheritance, making code more flexible and maintainable.

✅ Abstract Class in Java – Interview Answer:

An abstract class in Java is a class that cannot be instantiated and is declared using the abstract keyword. It may contain abstract methods (without implementation) as well as concrete methods (with implementation).

It serves as a base class and provides partial abstraction, allowing common behavior to be shared by all subclasses.

🔹 Key Features:

  • Can have both abstract and non-abstract methods

  • Can define constructors and instance variables

  • Cannot be instantiated directly

  • A subclass must override all abstract methods, unless it is also abstract

🔹 Syntax Example:

java
abstract class Animal {
    abstract void sound();  // abstract method

    void sleep() {          // concrete method
        System.out.println("Sleeping...");
    }
}

class Dog extends Animal {
    void sound() {
        System.out.println("Bark");
    }
}

🔹 Use Cases:

  • When you want to provide a common base with shared code and enforce method implementation

  • For partial abstraction

  • When all child classes should follow a template structure

🔚 Summary:

An abstract class provides a way to define a template for other classes with shared logic and enforced methods, making code more structured and reusable.
